# full-duplex-uart
A full-duplex UART transceiver implemented in SystemVerilog, with independent RX/TX, 9-bit packet framing, and even-parity error detection. Verified by using two UARTs to transmit a randomly generated byte to each other.

## Overview
This project implements a UART (Universal Asynchronous Receiver/Transmitter) capable of simultaneous, independent transmission and reception, including:

- Independent RX and TX finite state machines, allowing full-duplex operation (both directions active at once)
- 9-bit packet framing (8 data bits plus 1 even-parity bit)
- Even-parity error detection on receive, flagged via rx_error
- Handshake-gated transmission (tx_needed + peer rx_available) to prevent data collisions between simultaneous transmitters
- A configurable cycles_per_bit parameter, set low in simulation for waveform readability (not representative of a real baud rate)

## Architecture
```
+----------------------------+----------------------------------------------------+
|       Signal/Block         |                     Purpose                         |
+----------------------------+----------------------------------------------------+
| rx_state (4-state FSM)     | IDLE -> LOAD -> DECODE -> SEND TO DATABUS           |
+----------------------------+----------------------------------------------------+
| tx_state (5-state FSM)     | IDLE -> SEND DATA BITS -> PARITY BIT -> STOP BIT    |
|                            | -> REST                                             |
+----------------------------+----------------------------------------------------+
| current_rx_data            | Shift register accumulating received bits before    |
|                            | parity check and databus output                     |
+----------------------------+----------------------------------------------------+
| rx_error                   | Asserted if even-parity check fails or stop bit     |
|                            | is invalid                                          |
+----------------------------+----------------------------------------------------+
```

## How to Run
1. Compile uart.sv and testbench.sv in your SystemVerilog simulator
2. Run simulation
3. View output and waveforms. A screenshot of waveform is included in waveforms/, and output is included in docs/

## Verification
Verified by using two UARTs to transmit one randomly generated byte to each other per simulation run. The $display shows expected output, and actual output. There is a if statement that shows if either of the parity bits has failed.

## Design Notes/Known Limitations
- cycles_per_bit is manually set to 4 instead of using the formula of clock frequency/baud rate, as the otherwise incredibly low cycles_per_bit would make the waveform too long and messy
- Verification only involves a simultaneous full-duplex transmission of two randomly generated bytes
